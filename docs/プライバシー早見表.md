# 🔒 プライバシー早見表 — どのLLMを使うべき？

> バディに何を任せるかで「使っていいAI」は変わる。
> 「秘書に家計を任せる」なら学習されたら困るし、「公開ブログのネタ出し」ならどこ使っても問題ない。
> この資料は **判断材料の早見表** です。最終判断は自分の用途と照らして決めてね🌊

## 🤔 まず「なぜ気にするのか」

LLM API に送ったあなたのメッセージ・ファイルは、**サービス提供側の3つのうちどれか**で扱われます：

| パターン | 何が起きるか | 例 |
|---|---|---|
| 🟢 **完全パススルー** | 質問→回答だけ、ログも残らない | OpenRouter ZDR / Anthropic Claude / OpenAI 有料 |
| 🟡 **ログのみ保存** | 一定期間メタ/会話保存（法規制対応・誤用検知）、学習には使わない | 多くの有料API |
| 🔴 **学習に使われる** | 将来モデルの訓練データに混ざる可能性 | 多くの無料API（Gemini無料含む） |

**家計データ・健康データ・人間関係の悩み・パスワード・社内資料**を投げる前に、相手プロバイダがどれかは知っておきたい。

## 🎯 用途別 おすすめ早見表

### 家計データ・健康データ・パスワード等の機微情報
→ **Anthropic 有料 / OpenAI 有料 / Mistral / ローカルLLM**

### 一般的な会話・予定管理・要約・調べ物
→ **OpenRouter + training opt-out 設定 / Mistral / Gemini Flash**

### コードレビュー・ブログのネタ出し・公開予定の文章
→ **どれでもOK**（ただしコード内のシークレットには注意）

### 仕事の社内資料・顧客データ
→ **Anthropic / OpenAI 有料の Enterprise / ローカルLLM**
（無料ティアは絶対NG）

## 🛠️ OpenRouter で「学習されない設定」にする手順

OpenRouterは **アカウント設定で `Allow training` を OFF** にすると、自動的に「学習する provider」を routing から除外してくれる。

1. https://openrouter.ai/settings/privacy にアクセス
2. **「Enable input/output logging」** → **OFF**
3. **「Allow providers to train on your data」** → **OFF**
4. 保存

これで OpenRouter 経由で投げたデータは：
- OpenRouter 自体は ZDR（保存しない）
- 学習する provider は自動除外
- ログ取る provider も自動除外

**実質「ZDR + 学習なし」運用**になる。

### さらに中国系 provider を完全除外したい場合

API 呼び出し時に `provider` パラメータで `data_collection: 'deny'` や `order` で米国/EU プロバイダのみ指定できる。OpenClaw からの設定方法は別途。

## 🇨🇳 DeepSeek の話（よく聞かれるので独立セクション）

DeepSeek は **「モデル自体は中国製」だけど「インフラは選べる」** が分かれ目。

### DeepSeek 公式 API（直接アクセス）
- 中国（杭州 Hangzhou DeepSeek 社）にデータ送信される
- デフォルト学習、opt-out 可能
- **ただし「model training feedback loops」を consent 不要の例外条項に入れてる** → opt-out しても完全には逃れられない可能性
- 中国データ規制下にある = データ要請に応える可能性
- 推奨度: ❌ プライベートな用途には避ける

### OpenRouter 経由 DeepSeek :free
- モデルは中国製、インフラは provider 次第
- 西側プロバイダ（DeepInfra / Together AI / Fireworks 等）がホストすることが多い
- **OpenRouter で training opt-out 設定すれば、データは中国に行かない**
- 推奨度: ⭕ プライバシー設定すれば普段使いOK
