### 00:10-00:25 セットアップ

**説明しながら一緒に進める:**

1. リポダウンロード
   - GitHub にアクセス: `https://github.com/seika86/openclaw-starter`
   - 「Code」緑ボタン → 「Download ZIP」
   - ダウンロードフォルダで解凍 → デスクトップなど好きな場所に移動

2. PowerShell or ターミナルで cd
   ```bash
   cd Desktop/openclaw-starter   # または解凍した場所
   ```

3. `.env` 作成
   - **Windows:** `Copy-Item .env.example .env` → `notepad .env`
   - **Mac:** `cp .env.example .env` → `open -a TextEdit .env`

4. `.env` を編集
   - `GEMINI_API_KEY=` の右側にキーを貼る
   - `DISCORD_BOT_TOKEN=` の右側に Bot Token を貼る
   - **`OPENCLAW_GATEWAY_TOKEN=`** の右側に、ターミナルで `openssl rand -hex 32` を実行して出た文字列を貼る（後で Dashboard ログインに使うので残しておく）
   - 保存して閉じる

**詰まりポイント:**
- ⚠️ ZIP解凍後フォルダ名が `openclaw-starter-main` になることがある → そのままで OK
- ⚠️ `notepad .env` で「ファイルが見つかりません」 → `.env.example` のコピーを忘れてる
- ⚠️ パスにスペース or 日本語が含まれる → ターミナルで cd するとき注意（クォートで囲む）

### 00:25-00:35 起動

```bash
docker compose up -d
```

**説明:**
- 初回はインターネットからイメージをダウンロード（3〜5分）
- 進捗バーが何個か並んで動く
- 終わったら `Started` と表示される

**進捗確認:**
```bash
docker compose ps
```
- `(healthy)` になればOK
- `(starting)` ならもう少し待つ

**LLMプロバイダ登録（初回のみ）:**
```bash
docker compose exec buddy openclaw onboard \
  --non-interactive --accept-risk --mode local \
  --auth-choice gemini-api-key \
  --gemini-api-key "$GEMINI_API_KEY"

# Pro は無料枠で使えないので Flash に切替
docker compose exec buddy node -e "
const fs=require('fs'),p='/home/node/.openclaw/openclaw.json';
const c=JSON.parse(fs.readFileSync(p));
c.agents=c.agents||{};c.agents.defaults=c.agents.defaults||{};
c.agents.defaults.model={primary:'google/gemini-2.5-flash'};
fs.writeFileSync(p,JSON.stringify(c,null,2));console.log('OK');
"
docker compose restart
```

**ブラウザで開く:**
- http://localhost:18789

→ 「OpenClaw Gateway ダッシュボード」表示されたら **Gateway トークン** 欄に `.env` の `OPENCLAW_GATEWAY_TOKEN=` の右側の値をコピペ → 「接続」クリック

→ Dashboard が立ち上がれば成功！🎉

**詰まりポイント:**
- ⚠️ `port is already allocated` → `.env` で `OPENCLAW_GATEWAY_PORT=28789` に変えて再起動
  - **その後 origin 許可リストにも追加が必要**（READMEのトラブルシューティング参照）
- ⚠️ Docker Desktop が起動してない → クジラアイコン🐳確認
- ⚠️ pullがめちゃ遅い → 会場Wi-Fi混んでる、隣の人が終わるまで待つ or テザリング切替
- ⚠️ ブラウザで「**origin not allowed**」エラー → ポート変えた人がハマる、READMEの手順で `allowedOrigins` 追加

### 00:40-00:55 Discord 連携

Dashboard の「**Channels**」 → 「**Discord**」を開く

1. Bot Token 入力欄に `.env` から自動読み込みされてるか確認
2. されてなければ手動で貼る → 保存
3. 「Connect」ボタン → 緑になればOK

**詰まりポイント:**
- ⚠️ Bot がオフラインのまま → Privileged Intents 全部ON再確認
- ⚠️ Token 貼り間違え → 前後の空白チェック
- ⚠️ サーバー招待してない → README の「Discord Bot の作り方」末尾参照
- ⚠️ `Cannot have install fields on a private application` エラー → Public Bot OFF 前に Installation Link を None にする必要あり（順序問題）
- ⚠️ `.env` の Token を変更したのに反映されない → `docker compose up -d --force-recreate`（restart だと env 再読み込みされない）
- ⚠️ Bot が「openclaw pairing approve…」と言ってくる → コンテナ内で実行: `docker compose exec buddy openclaw pairing approve discord <コード>`
- ⚠️ CLI で `Discord requires token` エラー → `--token` フラグを使う（`--bot-token` ではない）

## 🆘 講師サイドのカンペ

### よくある質問への回答

**Q: お金かかる？**
A: Docker も OpenClaw も Gemini も無料枠で完結。継続的に使うなら Anthropic の月3000円が一番賢くなる。

**Q: バディの記憶は消える？**
A: PCの中のDocker volumeに残るので、PCを買い替えたり完全リセットしない限り残る。バックアップ方法は次の一歩資料に。

**Q: 個人情報漏れない？**
A: workspaceフォルダの中身は LLM API には送られる（質問に必要な分だけ）。Gemini無料枠は学習に使われるので、機密情報は Anthropic / OpenAI 有料版を使うのがオススメ。

**Q: スマホからは使える？**
A: Discord/LINEのスマホアプリから話しかければ OK。バディ本体は家のPCで動いてる。

**Q: PC を寝かせると止まる？**
A: スリープすると Docker も止まる。常時起動にしたい場合は「PC のスリープ無効化」or 「専用Mac mini を立てる」（Seikaがやってる構成）。
